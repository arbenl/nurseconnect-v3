# NC-E2-04 Threat Model

## Slice

NC-E2-04 / codex/medical-evidence-brand.

## Scope

Add proof-token guards for nurse credential status writes and clinical visit
medical evidence substrate. The slice composes with branded `PolicyDecision`
and `OrganizationId` tokens from NC-E2-03.

Non-scope: schema changes, RLS predicates, tenant backfill, route-wide session
migration, outbox conversion, clinical write routes, retention, erasure, and
field encryption.

## Assets

- Nurse credential status, verification, rejection, and suspension facts.
- Actor identity and tenant authorization proof used to mint evidence.
- Clinical visit evidence substrate for future PHI write paths.
- Admin audit rows co-committed with credential state changes.

## Trust Boundaries

- Admin handlers must derive actor and tenant context server-side.
- Domain constructors mint evidence only after platform authz proof validates.
- Persistence helpers accept branded updates, not caller-built status objects.
- Architecture guards block raw `nurses.status` writes outside allowlisted
  owning-domain constructors.

## STRIDE Findings

- Spoofing: caller-provided actor, tenant, or proof fields must not be trusted.
- Tampering: raw status writes and structural proof objects must fail.
- Tampering: stale proof must fail compare-and-set without changing state.
- Repudiation: credential changes must keep same-transaction admin audit rows.
- Information disclosure: proof and audit details must not carry PHI values.
- Denial of service: proof checks remain in-process and deterministic.
- Elevation of privilege: denied or wrong-tenant policy decisions must not mint
  credential or medical evidence.

## Residual Risk

`nurses` has no `organization_id` until NC-TB-01, so this slice binds tenant in
proof/authz context but cannot enforce a tenant predicate directly on that row.
Full PHI classification, read audit, encryption, and erasure remain NC-E5 work.

## Verification

- Type-level tests reject raw credential and medical evidence objects.
- Runtime tests reject plain-object, spread, and JSON proof forgeries.
- Tests prove stale proof conflicts and nurse-id mismatch is rejected.
- Architecture guard tests block unapproved `nurses.status` writes.
- `pnpm verify-slice -- --required-gates` records ent-gate evidence.
- Closeout PR #112 records PR #111 merge evidence and promotes NC-TB-01.
