# NC-E2-03 Threat Model

## Slice

NC-E2-03 / codex/platform-authz.

## Scope

Add in-process tenant/resource-aware policy functions plus phantom-type guards
for service request status transitions and branded policy decisions.

Non-scope: schema changes, RLS policy changes, production notifications,
clinical workflow changes, data retention/erasure behavior, and external
authorization services.

## Assets

- Tenant-scoped service request state.
- Actor identity, role, and organization membership facts.
- PHI-bearing request fields protected by policy decisions.
- Audit and event evidence for authorized transitions.

## Trust Boundaries

- Request handlers call platform policy functions before domain state changes.
- Domain transition constructors accept only authorized policy proof tokens.
- Tests and type checks prove denied, cross-tenant, and raw-status paths fail.

## STRIDE Findings

- Spoofing: actor identity must come from the platform identity boundary, not
  caller-provided ids.
- Tampering: raw string status writes must stay blocked outside owning
  transition functions.
- Repudiation: authorized transitions must preserve audit/event evidence.
- Information disclosure: policy decisions must deny wrong-tenant and PHI-field
  access by default.
- Denial of service: policy checks must be deterministic in-process logic.
- Elevation of privilege: admin, nurse, patient, and partner rules require
  negative tests for wrong role and wrong tenant.

## Residual Risk

The slice will improve in-process authorization, but shared-schema RLS,
outbox-backed audit semantics, and full CQRS read-model isolation remain
separate Phase C work.

## Verification

- Policy matrix tests for allow, deny, wrong-role, wrong-tenant, and PHI-field
  cases.
- Type-level negative test proving direct service request status writes fail
  without an AuthorizedTransition token.
- `pnpm verify-slice -- --required-gates` with ent-gates evidence for this file.
