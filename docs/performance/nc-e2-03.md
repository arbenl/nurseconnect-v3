updates. It does not add database queries, remote calls, background jobs,
route rewiring, bundle code, polling, or new patient-facing UI.

- `apps/web/src/server/requests/request-lifecycle.test.ts`

- one branded object allocation when a request transition is authorized
- one symbol-property runtime proof check before deriving the target status
- one additional `service_requests.status = expected` predicate in existing
